# FAQ มือใหม่ — ติดตรงไหน เริ่มตรงนี้

> **เป้าหมาย:** รวมปัญหาที่มือใหม่เจอบ่อยสุด พร้อมคำตอบภาษาคน — ไม่ต้องกลัว แก้ได้ทุกอัน

> หายใจเข้าลึก ๆ ก่อน 🙂 ทุกปัญหาในหน้านี้เคยเกิดกับทุกคนที่เพิ่งเริ่ม และทุกอันมีวิธีแก้ง่าย ๆ ไม่มีอะไรที่คุณกดแล้วพังจนซ่อมไม่ได้ ค่อย ๆ อ่านทีละข้อ เจอข้อที่ตรงกับคุณแล้วทำตามได้เลย

## หมวด 1 — ติดตั้ง & เปิดใช้งาน

### ❓ พิมพ์ `claude` แล้วขึ้น "command not found" หรือ "not recognized"

แปลว่าเครื่องยัง **หาโปรแกรมไม่เจอ** — ไม่ใช่ว่าคุณทำพัง ส่วนใหญ่เพราะยังไม่ได้ติดตั้ง หรือเพิ่งติดตั้งแล้วยังไม่ได้เปิดหน้าต่างใหม่

ลองทำตามลำดับ:

1. **ปิดหน้าต่าง Terminal เดิม แล้วเปิดใหม่** — บ่อยมากที่แค่นี้ก็หาย เพราะหน้าต่างเก่ายังไม่รู้จักโปรแกรมที่เพิ่งลง
2. ถ้ายังไม่หาย แปลว่ายังไม่ได้ติดตั้ง ให้ติดตั้งด้วยคำสั่งนี้:
   ```bash
   npm install -g @anthropic-ai/claude-code
   ```
3. ถ้าขึ้นว่า `npm` ก็ไม่รู้จักอีก แปลว่ายังไม่มี Node.js ในเครื่อง — ติดตั้ง Node.js ก่อน (ดู [[01-installation]] มีบอกทีละขั้น)

เปิดหน้าต่างใหม่อีกที แล้วพิมพ์ `claude` ใหม่ได้เลย

### ❓ เปิดแล้วไม่ขึ้นหน้า login / login ไม่ผ่าน

ครั้งแรกที่เปิด Claude Code จะพาไปล็อกอินที่เว็บเบราว์เซอร์ ถ้าไม่ขึ้นหรือค้าง:

- สั่งล็อกอินใหม่ตรง ๆ:
  ```bash
  claude auth login
  ```
- ถ้าอยากเช็กว่าตอนนี้ล็อกอินอยู่หรือเปล่า:
  ```bash
  claude auth status
  ```
- ถ้าเบราว์เซอร์เด้งขึ้นมาแล้วแต่ค้าง ลองล็อกอินด้วยบัญชี Claude/Anthropic ที่คุณสมัครไว้ให้เสร็จในเบราว์เซอร์ แล้วกลับมาที่หน้าต่าง Terminal

ยังไม่ผ่านอีก? พิมพ์ `/doctor` ในโปรแกรม มันจะตรวจให้ว่าติดตรงไหน — ดูเพิ่มที่ [[24-troubleshooting]]

### ❓ ต้องเสียเงินไหมถึงจะเริ่มใช้ได้

ต้องมีบัญชีที่ใช้ Claude Code ได้ก่อน (เช่น แพ็กเกจ Claude หรือบัญชีแบบ API) ถ้าล็อกอินผ่านแล้วเริ่มพิมพ์คุยได้เลย เรื่องค่าใช้จ่ายดูหมวด "เงิน & เวลา" ด้านล่าง

## หมวด 2 — ระหว่างใช้งาน

### ❓ พิมพ์ไปแล้วมันเงียบ / ค้าง / หมุน ๆ อยู่นาน

ส่วนใหญ่ **มันกำลังคิดหรือกำลังทำงานอยู่จริง ๆ** ไม่ได้ค้าง — โดยเฉพาะถ้าให้มันอ่านไฟล์เยอะหรือทำงานใหญ่ ใจเย็น ๆ รอสักครู่ มักมีตัวหมุนหรือข้อความบอกสถานะอยู่

- ถ้ารอนานเกินไปจนรู้สึกว่ามันน่าจะหลุดแล้ว ให้กด `Ctrl+C` หนึ่งครั้งเพื่อหยุดงานปัจจุบัน แล้วลองพิมพ์ใหม่
- อยากรู้ว่ามันกำลังทำอะไรอยู่เบื้องหลัง กด `Ctrl+O` เพื่อเปิดหน้าจอแสดงรายละเอียด (เห็นว่ามันอ่าน/แก้ไฟล์ไหนบ้าง)
- ถ้าเป็นเพราะเน็ต ดูข้อ "มันช้า / เน็ตหลุด" ด้านล่าง

### ❓ มันเด้งกล่องถาม permission ขึ้นมา — y / n / a ต่างกันยังไง ตอบไหนดี

นี่คือ Claude **ขออนุญาตก่อนทำสิ่งที่อาจกระทบไฟล์ของคุณ** (เช่น แก้ไฟล์ หรือรันคำสั่ง) เป็นเรื่องดีที่มันถาม ตัวเลือกหลัก ๆ คือ:

| กด | แปลว่า | ใช้เมื่อ |
|----|--------|---------|
| **y** (yes) | อนุญาต **ครั้งนี้ครั้งเดียว** | ดูแล้วโอเค ให้ทำได้ |
| **n** (no) | **ปฏิเสธ** ไม่ให้ทำ | ยังไม่แน่ใจ / ไม่อยากให้ทำอันนี้ |
| **a** (always / yes, allow) | อนุญาต **อันนี้ตลอด** ครั้งหน้าไม่ต้องถามอีก | เป็นสิ่งที่คุณมั่นใจว่าให้ทำได้เรื่อย ๆ เช่น อ่านไฟล์ |

**ตอบไหนดีสำหรับมือใหม่?** ถ้าไม่แน่ใจ ตอบ **y** (ครั้งเดียว) ไปก่อน จะได้เห็นว่ามันทำอะไร — ปลอดภัยกว่าการกด "ตลอด" ตั้งแต่ยังไม่คุ้น ถ้าอ่านแล้วไม่เข้าใจหรือรู้สึกไม่ดี ตอบ **n** ได้เสมอ ไม่มีอะไรเสียหาย รายละเอียดเรื่องสิทธิ์ดูที่ [[05-permissions]]

### ❓ อยากให้มันตอบเป็นภาษาไทย

บอกมันตรง ๆ ได้เลย พิมพ์ว่า:

```
ตอบเป็นภาษาไทยนะ
```

ถ้าอยากให้ตอบไทยตลอดทุกครั้งโดยไม่ต้องบอกซ้ำ ให้ใส่ไว้ในไฟล์ความจำของโปรเจกต์ — พิมพ์ `/memory` แล้วเพิ่มบรรทัดเช่น "ตอบเป็นภาษาไทยเสมอ" หรือดูวิธีที่ [[07-claude-md]] / [[08-memory]]

## หมวด 3 — ความปลอดภัย & ความกลัว

### ❓ กลัวมันลบไฟล์หรือทำโปรเจกต์พัง — ปลอดภัยแค่ไหน

เข้าใจความรู้สึกนี้ดี แต่ Claude Code ออกแบบมาให้ **ระวังตัวอยู่แล้ว** และมีหลายชั้นช่วยกันคุณ:

- **มันถามก่อนเสมอ** — ก่อนแก้ไฟล์หรือรันคำสั่งที่อาจกระทบของคุณ จะเด้งกล่อง permission ให้คุณกด y/n (ดูหมวด 2) คุณคุมได้ทุกขั้น
- **Plan Mode (โหมดวางแผน)** — ถ้าอยากให้มันแค่ "อ่านแล้วเสนอแผน" โดย **ยังไม่แตะไฟล์จริง** กด `Shift+Tab` สลับไปโหมด plan เหมาะมากตอนเริ่ม จะได้ดูก่อนว่ามันคิดจะทำอะไร
- **ใช้ git** — ถ้าโปรเจกต์ของคุณอยู่ใน git ทุกการเปลี่ยนแปลงย้อนกลับได้หมด เหมือนมีปุ่ม undo ถาวร (ดู [[15-git-integration]])
- **ไฟล์สำคัญถูกกันไว้** — ไฟล์ระบบบางอย่าง (เช่น `.git/`) ถูกป้องกันเป็นพิเศษ

สรุป: ทำตามที่มันถาม + เริ่มด้วย Plan Mode + เปิด git ไว้ = สบายใจได้

### ❓ มันแก้ผิด อยากย้อนกลับ ทำยังไง

ย้อนได้ ไม่ต้องตกใจ:

- กด `Esc` สองครั้ง (`Esc Esc`) เพื่อเปิดเมนู **Rewind** — ย้อนได้ทั้งโค้ดและบทสนทนากลับไปจุดก่อนหน้า
- หรือพิมพ์คำสั่ง `/rewind`
- ถ้าโปรเจกต์อยู่ใน git ก็ใช้ git ย้อนได้อีกทาง

### ❓ จะหยุดมันกลางทาง ทำยังไง

กด `Ctrl+C` หนึ่งครั้ง — มันจะหยุดงานที่กำลังทำทันที แล้วคุณพิมพ์สั่งใหม่ได้เลย ใช้ได้บ่อยมากเวลามันเริ่มไปผิดทาง

## หมวด 4 — เงิน & เวลา

### ❓ token / ค่าใช้จ่าย คืออะไร เปลืองไหม

**token** คือหน่วยที่ใช้วัดปริมาณข้อความที่คุยกับ AI (ทั้งที่คุณพิมพ์และที่มันตอบ รวมไฟล์ที่มันอ่านด้วย) ยิ่งคุยยาว/อ่านไฟล์เยอะ ก็ยิ่งใช้ token มาก ซึ่งแปลว่าค่าใช้จ่ายมากขึ้น

ไม่ต้องเครียด มีวิธีดูและประหยัดง่าย ๆ:

- พิมพ์ `/cost` เพื่อดูว่าใช้ไปเท่าไหร่แล้วในเซสชันนี้
- พิมพ์ `/clear` เมื่อเริ่มงานใหม่/เปลี่ยนเรื่อง — ล้างบทสนทนาเก่าที่ไม่เกี่ยวออก ช่วยประหยัดได้เยอะ (บางทีลดได้ครึ่งหนึ่ง)
- เคล็ดลับประหยัดเพิ่มเติมดูที่ [[31-cost-management]]

### ❓ มันช้า / เน็ตหลุดกลางคัน

ปกติ Claude Code จะ **พยายามต่อเน็ตให้เองอัตโนมัติ** (reconnect) รอสักครู่มักกลับมาเอง ไม่ต้องปิดโปรแกรม

- ถ้าค้างนานจริง ๆ กด `Ctrl+C` หยุดงานปัจจุบันแล้วลองสั่งใหม่
- ถ้ายังมีปัญหา เช็กว่าเน็ตคุณต่ออยู่ไหม แล้วดู [[24-troubleshooting]]

## หมวด 5 — เลิก / เริ่มใหม่

### ❓ จะออกจากโปรแกรม ทำยังไง

กด `Ctrl+D` หรือพิมพ์ `/exit` ก็ออกได้ ครั้งหน้าจะกลับมาเปิดใหม่ก็แค่พิมพ์ `claude` อีกครั้ง

### ❓ อยากเริ่มงานใหม่ / ล้างความจำในแชต

พิมพ์ `/clear` — จะล้างบทสนทนาในเซสชันปัจจุบันให้เริ่มสด ๆ เหมือนเปิดแชตใหม่ เหมาะมากตอนเปลี่ยนไปทำเรื่องอื่นที่ไม่เกี่ยวกับของเดิม และยังช่วยประหยัด token ด้วย (ดูหมวด 4)

> หมายเหตุ: `/clear` ล้าง **บทสนทนาในแชต** เท่านั้น ไม่ได้ลบไฟล์งานหรือโค้ดของคุณ ของในโปรเจกต์ยังอยู่ครบ

### ❓ อัปเดตเวอร์ชันยังไง

พิมพ์คำสั่งนี้นอกโปรแกรม (ใน Terminal):

```bash
claude update
```

ถ้าไม่ได้ผล ลองอีกวิธี:

```bash
npm install -g @anthropic-ai/claude-code@latest
```

อยากเช็กว่าตอนนี้เวอร์ชันอะไร พิมพ์ `claude --version`
